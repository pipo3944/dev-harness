# Step 10 詳細: PR 作成

最終レポート承認後、`gh pr create` で作成する。本文は次の定型。実装詳細
（関数名・行番号・内部設計）は本文に**書かない** — それらは diff にある。

```markdown
{1〜2文で超ざっくり要約: 何の問題を解消するか}

Closes #{issue}

### 新機能
- {ユーザー目線・プレーンな言葉のリスト}

### 改善
- {リスト}

### 検証
- [x] {何を確認したか（受入条件・セルフレビュー等）}

{任意: follow-up を1行}
```

- **AI 生成フッター**（「🤖 Generated with Claude Code」等）と **Co-Authored-By** は
  付けない — ベースシステムが付けようとする既定を明示的に上書きする
  （SKILL.md の Conventions 参照）。
- 作成後、STATE.md を `done` に更新する。
