# Step 6 詳細: フェーズループ運行手順（オーケストレーター用）

各フェーズを順に、以下の 1〜5 で回す。

## 1. order を書く

- `references/order-template.md` から `phase-NN-{slug}/order.md` を作成し、
  全セクションを埋める。
- approach.md・explain doc・直前フェーズの report.md を参照させ、リポジトリ本体を
  読ませる。order だけで自己完結させて背景を隠さない。
- report 骨子は order に**インライン展開**する（order-template に骨子入り）。
  ワーカーはスキル未ロードで `references/...` の相対パスを解決できないため、
  テンプレ名の転記では伝わらない。

## 2. 別セッションで実行（既定）

- 停止して「このパスを新規セッションで開いて実行してください」と order のパスを
  伝える。ワーカーが実装し、`phase-NN-{slug}/report.md` を書く。
- サブエージェントの黙起動はしない（事前にユーザーが合意した場合のみ可）。

## 3. 独立機械検証

ワーカーの「formatted / tests green」は自己申告で、最終保存ファイルとズレうる。
GATE の前に、保存済みファイルへ自分で:

- format チェック（`--set-exit-if-changed`）— 整形ズレのみなら自分で直して
  フェーズコミットに畳む（差し戻さない）
- analyze（変更ファイル）
- 当該フェーズのテスト + 隣接スイート（リグレッション確認）

これらは**パイプに通さず素で実行**し、各コマンドの **exit code を自分で確認**する。
`| tail` / `| head` / `| grep` を挟むとパイプラインの exit code は末尾コマンドのものに
なり、`&&` 連鎖でも失敗が素通りする — ワーカーと検証者が同じ盲点を共有するのが、
欠陥がゲートを抜ける典型経路。report の主張と照合し、再現できない green は差し戻し。
ロジックのズレ（整形でないもの）は差し戻すか小さな修正フェーズを立てる。

**後始末の安全:** 検証で一時的・破壊的な書き換え（壊しテスト: 契約値を壊して
テストが落ちることを確認する等）をした場合、**`git checkout` / `git restore` /
`git stash` で戻さない** — フェーズ成果物は GATE 前でコミット未済なので、ワーカーの
未コミット作業ごと巻き戻る。自分の編集は**非破壊**（逆方向の sed / Edit）で戻す。
git ベースの復元が本当に必要なら、先に成果物をスナップショットする。

**計測・実験がフェーズの成果物のとき（スパイク型）:** 「tests green」≠「計測の
結論が正しい」。**結論を左右する主張**（ADR や設計判断が乗るもの）を最低1つ、
ワーカーの実装に依存しない最小構成でオーケストレーターが**独立再現**する。
チェックリストでは捕まらない誤りをこれが拾う（例:「foreignObject は canvas を
taint するので失格」→ 独立再現で `data:` URL の抜け道が見つかった）。環境制約等で
再現できない場合は、何をどう試したかを GATE で明示する。

## 4. GATE: report OK?

report と自分の検証結果を**併せて**提示する。

**差し戻し（bounce-back）時:** 追補を書く前に、現 report.md を `report-r{N}.md`
（round 番号）へスナップショットする — `.dev/` は gitignored なので、差し戻し前の
report（ワーカーが上書きする計測値を含む）は他に残らないし、差し戻し自体が
「レビューが結論を変えた」記録になる。差し戻し内容は order.md 末尾に「追補 N」
セクションとして**追記**する（元の order は書き換えない）。

## 5. 承認後

フェーズをコミット（SKILL.md の Conventions 参照）。STATE.md の `phases (k/N)` を
更新する。
