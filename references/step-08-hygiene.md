# Step 8 詳細: Pre-PR hygiene チェックリスト

PR 前に累積差分をこのチェックリストで一掃する。

1. **運行内部の記号がコメントに残っていないか** — レビュー finding ID（`M9`/`N2`）、
   案名（`案B`）、フェーズ番号など、この運行でしか意味を持たない記号。
   grep 種 = **固定パターン + STATE.md の内部記号レジストリ**:
   `（[MN][0-9]+）`・`案[A-Z]`・`Phase ?[0-9]` から始め、レジストリの登録記号ぶん
   パターンを足す（固定パターンだけでは `Q1c`/`U1〜U3` のような運行発明の記号を
   取りこぼす）。ヒットは意味で判断し、今回タスクの差分だけを触る。残すべき説明は
   記号でなく**理由ベース**に書き換える。
2. **過剰コメント**（自明の言い換え）→ 削る。
3. **コメント不足** — 自明でない不変条件や「なぜ」に説明が要る箇所 → 足す。
4. **テキスト/markdown の欠陥**:
   - 文字化け（U+FFFD）: `LC_ALL=C grep -nP $'\xef\xbf\xbd' {対象ファイル}`
   - markdownlint（例: MD040 フェンスの言語指定漏れ）
   - 過剰断定（「確実に成功する」→ 軟化）

出力は**挙動・成果物の変更なし**の単独コミット（例: `docs(scope): コメント棚卸し`）。
オーケストレーター直またはライトなサブエージェントで実施。GATE は設けず、結果は
最終レポートで見せる。
